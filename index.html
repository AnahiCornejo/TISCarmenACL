Gestión Hotelera

Empleados (Personal)

Datos clave para la base de datos:

  • ID_Empleado: Identificador único del empleado (clave primaria).
  • Nombre, Apellido_Paterno, Apellido_Materno: Información personal del empleado.
  • Puesto: Cargo o función en la empresa.
  • Turno: El turno asignado.
  • Hora_Entrada, Hora_Salida: Horarios específicos.
  • Sueldo: Remuneración del empleado.
  • Fecha_Ingreso: Fecha en que el empleado comenzó a trabajar.
  • Telefono: Número de contacto.
  • Observaciones: Notas adicionales.

Administrador

Permisos:

  • Crear, actualizar y eliminar empleados.
  • Ver y modificar todos los datos de los empleados:
    • ID_Empleado
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Puesto
    • Turno
    • Hora de Entrada
    • Hora de Salida
    • Sueldo
    • Fecha de Ingreso
    • Teléfono
    • Observaciones

Gerente

Permisos:

  • Ver y actualizar la información de los empleados bajo su supervisión.
  • Campos que puede modificar:
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Puesto
    • Turno
    • Hora de Entrada
    • Hora de Salida
    • Fecha de Ingreso
    • Teléfono
    • Observaciones

Restricciones:

  • No puede modificar el Sueldo.
  • No puede eliminar empleados.
  • No puede modificar el ID_Empleado.

Recepcionista

Permisos:

  • Ver información básica de los empleados:
    • ID_Empleado
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Puesto
    • Teléfono

Restricciones:

  • No puede ver ni editar: Turno, Hora de Entrada, Hora de Salida, Sueldo, Fecha de Ingreso, Observaciones.
  • No puede eliminar empleados.

Diagrama de la Base de Datos

Tablas

CREATE TABLE Empleados_Basico (
    ID_Empleado INT IDENTITY(1,1) PRIMARY KEY,
    Nombre VARCHAR(100) NOT NULL,
    Apellido_Paterno VARCHAR(100) NOT NULL,
    Apellido_Materno VARCHAR(100) NOT NULL,
    Puesto VARCHAR(50) NOT NULL,
    ID_Horario INT NULL,
    Fecha_Salida DATE NULL
);

CREATE TABLE Empleados_Sueldo (
    ID_Empleado INT PRIMARY KEY,
    Sueldo DECIMAL(10,2) NOT NULL,
    Fecha_Ingreso DATE NOT NULL,
    FOREIGN KEY (ID_Empleado) REFERENCES Empleados_Basico(ID_Empleado)
);

CREATE TABLE Empleados_Contacto (
    ID_Empleado INT PRIMARY KEY,
    Telefono NVARCHAR(15),
    Correo VARCHAR(100) NULL,
    Observaciones NVARCHAR(255),
    FOREIGN KEY (ID_Empleado) REFERENCES Empleados_Basico(ID_Empleado)
);

Clientes

Datos clave para la base de datos:

  • ID_Cliente: Identificador único del cliente.
  • Nombre, Apellido_Paterno, Apellido_Materno: Datos personales del cliente.
  • Fecha_Registro: Fecha en que el cliente se registró.
  • Tipo_Cliente: Clasificación del cliente (frecuente, ocasional, etc.).
  • Notas: Notas adicionales sobre el cliente.

Administrador

  • Crear, actualizar, eliminar y consultar toda la información de los clientes.
  • Acceso completo a todos los campos:
    • ID_Cliente
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Fecha_Registro
    • Tipo_Cliente
    • Notas
    • ID_Contacto
    • Tipo_Contacto
    • Valor
    • ID_Direccion
    • Tipo_Direccion
    • Calle
    • Ciudad
    • Estado
    • Codigo_Postal
    • Pais

Gerente

  • Ver y actualizar la información de los clientes.
  • Campos que puede modificar:
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Fecha_Registro
    • Tipo_Cliente
    • Notas
    • ID_Contacto
    • Tipo_Contacto
    • Valor
    • ID_Direccion
    • Tipo_Direccion
    • Calle
    • Ciudad
    • Estado
    • Codigo_Postal
    • Pais
  • Restricciones: No puede eliminar clientes ni crear nuevos clientes.

Recepcionista

  • Ver información básica de los clientes:
    • ID_Cliente
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Tipo_Cliente
  • Restricciones: No puede modificar ni eliminar clientes ni ver detalles completos.

Dueño

  • Crear, actualizar, eliminar y consultar toda la información de los clientes.
  • Acceso completo a todos los campos (igual que Administrador).

Diagrama de la Base de Datos

Tablas

CREATE TABLE Clientes (
    ClienteID INT IDENTITY(1,1) PRIMARY KEY,
    Nombre NVARCHAR(100) NOT NULL,
	Apelli_Paterno NVARCHAR(100) NOT NULL,
	Apellido_Materno NVARCHAR(100) NOT NULL,
    FechaRegistro DATETIME DEFAULT GETDATE(),
    TipoCliente NVARCHAR(50),
    Notas NVARCHAR(250)
);


CREATE TABLE Contactos_Cliente (
    ContactoID INT IDENTITY(1,1) PRIMARY KEY,
    ClienteID INT NOT NULL,
    TipoContacto NVARCHAR(20) NOT NULL, -- 'Telefono' o 'Correo'
    Valor NVARCHAR(50) NOT NULL,
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ClienteID)
);


CREATE TABLE Direcciones_Cliente (
    DireccionID INT IDENTITY(1,1) PRIMARY KEY,
    ClienteID INT NOT NULL,
    TipoDireccion NVARCHAR(20), -- 'Fiscal' o 'Residencial'
    Calle NVARCHAR(100),
    Ciudad NVARCHAR(50),
    Estado NVARCHAR(50),
    CodigoPostal NVARCHAR(10),
    Pais NVARCHAR(50),
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ClienteID)
);

-- Tabla de Datos Fiscales (opcional)
CREATE TABLE Clientes_Fiscal (
    ClienteID INT PRIMARY KEY,
    RFC NVARCHAR(15) NOT NULL,
    RazonSocial NVARCHAR(100),
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ClienteID)
);

Facturación

Datos clave para la base de datos:

  • ID_Factura: Identificador único de la factura.
  • Nombre_Razon_Social, RFC, Direccion_Fiscal, Telefono, Correo.
  • Fecha_Emision, Folio_Factura, Concepto, Subtotal, IVA, Total.

Administrador

  • Crear, editar, eliminar y consultar todas las facturas.
  • Asignar folios y modificar todos los campos.
  • Restricciones: Ninguna

Gerente

  • Ver y editar Subtotal, IVA y Total.
  • Restricciones: No puede crear, eliminar ni asignar folios.

Recepcionista

  • Crear nuevas facturas con información básica (Nombre_Razon_Social, RFC, Concepto, Telefono, Correo, Fecha_Emision).
  • Restricciones: No puede editar montos ni asignar folios.

Encargado de Facturación

  • Editar Fecha_Emision y Concepto.
  • Restricciones: No puede crear, eliminar ni modificar montos ni datos del cliente.

Dueño

  • Acceso completo a todos los campos, igual que Administrador.
  • Restricciones: Ninguna

Diagrama de la Base de Datos

Tablas

-- Tabla de clientes
CREATE TABLE Clientes_Info (
    ClienteID INT IDENTITY(1,1) PRIMARY KEY,
    Nombre_RazonSocial NVARCHAR(100) NOT NULL,
    Telefono NVARCHAR(15),
    Correo NVARCHAR(50)
);
GO

-- Tabla de datos fiscales
CREATE TABLE Clientes_Fiscal (
    ClienteFiscalID INT IDENTITY(1,1) PRIMARY KEY,
    ClienteID INT NOT NULL,
    RFC NVARCHAR(15) NOT NULL,
    DireccionFiscal NVARCHAR(150),
    TipoCliente NVARCHAR(20) NOT NULL,
    CONSTRAINT FK_Clientes_Fiscal_Clientes_Info FOREIGN KEY (ClienteID)
        REFERENCES Clientes_Info(ClienteID)
);
GO

-- Tabla de facturas
CREATE TABLE Facturas (
    FacturaID INT IDENTITY(1,1) PRIMARY KEY,
    ClienteID INT NOT NULL,
    FechaEmision DATETIME NOT NULL DEFAULT GETDATE(),
    FolioFactura NVARCHAR(50),
    Concepto NVARCHAR(255),
    Subtotal DECIMAL(10,2) NOT NULL,
    IVA DECIMAL(10,2) NOT NULL,
    Total DECIMAL(10,2) NOT NULL,
    CONSTRAINT FK_Facturas_Clientes_Info FOREIGN KEY (ClienteID)
        REFERENCES Clientes_Info(ClienteID)
);
GO
